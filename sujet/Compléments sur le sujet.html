Compléments sur le sujet

Quelques recommandations :

  1. l'objectif n'est pas de produire un site du niveau d'une application Java ! Il s'agit juste de mettre en oeuvre les bases de la conception Internet dans un cadre "réaliste" ;
  2. commencez toujours par des versions simples : cela sera plus facile à tester et vous êtes plus sûrs de vous pour ajouter des éléments ensuite ;
  3. vous pouvez ajouter des hypothèses de simplification dans ce que vous réalisez (tant qu'elles ne sont pas trop en contradiction avec le sujet, en cas de doute... un mail à votre chargé de TP) ;
  4. faîtes à votre niveau : construisez un site basé sur les éléments que vous avez correctement compris ;

Voici un exemple d'organisation de la BD pour un site d'e-commerce, dont vous pouvez vous inspirer pour dériver la structure nécessaire pour le sujet de cette année :

  • fichier des ressources : index, nom, description, caractéristiques...
  • fichier des réservations : index, nom ou index du produit, nom ou index du client, date de début, durée, quantité, commentaires...
  • fichier des clients : index, nom, prénom, coordonnées, courriel...
  • [fichier des utilisateurs autorisés : index, nom, prénom, qualité, courriel...] (facultatif, si gestion des authentifications)

La base de données sera simulée par (au moins) 3 fichiers texte sur le serveur (pas de base MySQL ou autre, pour éviter les problèmes de serveur). Deux fichiers serviront à mémoriser les informations relatives aux :

  • ressources
  • réservations
  • clients

Dans tous les cas, l'index est un numéro unique (au sein de chaque fichier) permettant d'identifier l'enregistrement correspondant. On n'utilise pas directement sa place dans le fichier (numéro d'ordre) afin de permettre les modifications (comme les suppressions) dans le fichier. Cet index sera utilisé dans le dernier fichier pour référencer un enregistrement (et donc sans être obligé de répéter toutes les données qui lui sont associées).

Attention : la partie authentification sur le site, bien qu'arrivant en premier dans l'ordre logique des fonctionnalités, est certainement la plus complexe. Il est fortement conseillé de la traiter en dernier.

Extensions : une fois les fonctions précédentes réalisées, tout ajout pourra être envisagé dans la mesure où son apport au site est justifié.
